# Novel Forge

一个离线可用的小说冲突生成器。给它几个关键词，它会把散乱的灵感整理成一条有因果关系的故事链，再把同一个源头事件推演成不同选择和不同后果。

它不是 AI 写作工具，更像一只认真递便签的灵感小助手：随机，但有规则；轻松，但讲逻辑。

## 可以做什么

- 输入关键词、主角身份、题材、叙事焦点和冲突强度。
- 生成一条完整逻辑链：源头、欲望、阻碍、行动、后果、代价、揭示、反转、结局状态。
- 用规则引擎和权重随机系统生成内容，不依赖云端 AI。
- 查看分支时间线：同一源头，不同选择，不同结局。
- 点击想法卡片，把灵感继续塞回输入区。
- 通过西施惠旁边的灵感气泡快速加入关键词。
- 每次生成后，新关键词会自动加入本地灵感池，去重保存。
- 支持复制、导出文本、历史记录和趋势分析。

## 如何使用

直接用浏览器打开：

```text
NovelForge.html
```

如果部署到 GitHub Pages，打开仓库页面即可，`index.html` 会自动进入主应用。

## 生成逻辑

项目内部使用三层非 AI 生成系统：

1. **规则引擎**
   根据关键词识别地点、人物、物件、秘密、危机等类型，再结合题材、焦点和强度筛选素材。

2. **故事语法树**
   固定生成链路：
   `源头 -> 欲望 -> 阻碍 -> 行动 -> 后果 -> 代价 -> 揭示 -> 反转 -> 结局状态`

3. **随机权重系统**
   每个素材都有标签和权重。系统不是完全随机，而是优先选择更符合当前输入的素材。

## 本地数据

这个项目不需要后端，也不会把输入内容上传到服务器。以下内容只保存在浏览器本地：

- 草稿输入
- 历史记录
- 趋势统计
- 灵感气泡词库

清空浏览器站点数据后，这些内容会被删除。

## 灵感气泡

首页角色旁边的气泡会随着创作变聪明：

- 默认显示几个初始灵感词。
- 每次生成后，输入框里的关键词会加入气泡池。
- 自动去重。
- 最新词优先显示。
- 最多显示 8 个气泡，最多保存 18 个本地灵感词。

## 一个小小的使用建议

不要只输入一个词。给它三到五个彼此有张力的词，效果会更好：

```text
海边小镇、失忆医生、旧灯塔、匿名病历、暴雨
```

故事通常不是从“好点子”开始的，而是从几个彼此不肯让步的词开始的。
